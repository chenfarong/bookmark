一、概述


账号系统和充值系统一样，都仅仅是提供给游戏服务器接入使用的服务器。对玩家是透明的。通信协议支持HTTP和WEBSOCKET两种方式。
游戏服务器和账号系统之间的合法验证方式HTTP采用cookies提供值（此值谨慎保存），WEBSOCKET握手成功后，先使用用户名加密码 登录后发送后续消息。

一、流程
0）支付前动作
1）客户端向服务器提交第三方票据
2）服务器向第三方验证票据有效性
3）通知游戏服务器执行游戏内购逻辑
4）充值成功后动作（BI统计数据）


二、系统概述
0）支付前动作，根据不同平台各不相同
1）客户端与游戏服务器交互
2）服务器与支付平台交互
3）支付平台与第三方平台交互


三、客户端与游戏服务器交互API
0）支付前动作【GIAP_BILL_BUY】
1）提交票据 【GIAP_BILL_TICKET】
2）查询票据处理结果【GIAP_BILL_QUERY】

说明
查看和设置都采用此协议，并且都必须提供一种验证方式，验证通过后将返回账号信息
请求
{
    "cmd":"GX_DSID_ACTOR",
    "auth":{
        "n":"15901937386",
        "t":"phone",
        "v":"123456"
    },
    "game":{
        "set":"jh|ios|xxx-sfsa-sadfasd",
        "unset":""
    }
}
响应
{
    "cmd":"GX_DSID_INFO_R",
    "rc":505,
    "msg":"验证没通过"
}
错误
- 303 协议错误
- 505 验证失败
- 606 设置或删除信息失败
- 0 操作成功
其他说明



    账号绑定角色 GX_DSID_ACTOR

    账号绑定角色 GX_DSID_ACTOR2


说明
支持解绑，【*游戏不提供此功能，GM可用】

pull 内容就是从game数组中删除
t的取值内容，和 GX_DSID_AUTH 相同
如果rc不存在也就是没有错误
